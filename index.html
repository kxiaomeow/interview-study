APIs — from zero to PM-fluent
01APIs for dummies — the essential mental modelStart here▼
The simplest true explanation
An API (Application Programming Interface) is a contract between two pieces of software that defines how they talk to each other. Think of it like a restaurant menu: the menu tells you what you can order (the available requests), what information you need to provide (the parameters), and what you'll get back (the response). You don't need to know how the kitchen works — just what's on the menu and how to order.

In Frame.io's context: when a third-party tool like ShotGrid wants to know "has this asset been approved in Frame.io?", it sends an API request to Frame.io's server. Frame.io processes it and sends back a response: "Yes, approved at 2:34pm by Jane Smith." The third-party tool never touches Frame.io's database directly — the API is the controlled interface between them.
The four things every API does (CRUD)
C
Create — POST
Add something new. "Create a new project in Frame.io." "Upload a file." "Add a comment."
R
Read — GET
Retrieve something. "Get the approval status of this asset." "List all files in this project." "Get all comments on this video."
U
Update — PUT / PATCH
Modify something. "Mark this asset as approved." "Update the metadata on this file." "Change the project name."
D
Delete — DELETE
Remove something. "Delete this version." "Remove a collaborator from a project."
REST vs. other types — what you need to know
Frame.io uses a REST API (Representational State Transfer). REST is the most common API style — it uses standard HTTP methods (GET, POST, PUT, DELETE) and returns data usually as JSON (a structured text format). Think of it as the universal language most web services speak.

The alternative you'll hear about: GraphQL — a query language where the client specifies exactly what data it wants (vs. REST which returns whatever the server decides to return). Frame.io uses REST. Worth knowing the distinction exists but you don't need to go deep.

1

What data and actions should be accessible through the API? In Frame.io: should third parties be able to trigger Firefly Actions? Read approval history? Write metadata? Every exposure decision is a product decision with security, privacy, and partnership implications.

03What "agentic AI" actually means — the working definitionEssential▼
Generative AI vs. Agentic AI — the key distinction
Generative AI responds to input. You ask it something, it produces an output. Every interaction starts fresh. It's reactive and single-step: prompt → response → done.

Agentic AI pursues goals. It takes multi-step actions, uses tools (APIs, databases, apps), makes decisions about what to do next, and can operate across multiple sessions with persistent memory. The key difference: an agent can be told "localize this campaign video into 5 languages and route the outputs to the right regional reviewers in Frame.io" — and it figures out all the steps to accomplish that goal without you specifying each one.

The practical test: does the AI take actions on your behalf, or does it just produce content for you to act on? Agentic = takes actions. Generative = produces content.
The 4 levels of agentic AI — know where things sit today
L1
Assisted — AI suggests, human decides and acts
Current state for most tools. Copilot, autocomplete, recommendations. Frame.io's NLP search and AI-suggested tags are here. Human is always in the loop for every action.
L2
Automated — AI acts on specific, bounded tasks
The Firefly Actions in Frame.io (Reframe, Dub, Remove Background) are here. Human triggers the action, AI executes it within a defined scope. Low risk because the scope is constrained.
L3
Orchestrated — AI coordinates multi-step workflows across tools
The Firefly AI Assistant is here, in beta. Human gives a high-level goal, AI figures out the steps and coordinates across apps. Human stays in the loop at key decision points but doesn't micromanage each step. This is where most of the interesting product design is happening right now.
L4
Autonomous — AI operates independently over extended periods
Not here yet in creative tools. Only 11% of enterprises run agentic AI in production — the gap between "experimenting" and "production" reflects how hard governance, trust, and accountability are at this level.
The key PM design questions for agentic products
1. Where does the human stay in control? The hardest design question. Every agentic feature needs a clearly defined handoff model — what decisions belong to the AI and what requires human confirmation. Getting this wrong destroys trust faster than not having AI at all.

2. What's the accountability model when the agent gets it wrong? If a Firefly agent applies the wrong feedback to a campaign video and it goes to a client, who's accountable? The PM, the product, the user who set up the agent? This needs to be designed explicitly — attribution, rollback, audit trail.

3. What's the memory model? An agent that starts from zero every session is limited. An agent that learns from past interactions — what got approved, what feedback patterns exist, what brand guidelines apply — compounds in value over time. Frame.io's approval history is the data that enables this memory.

4. How does the agent interact with other humans in the workflow? This is Victoria's specific framing — person-to-person collaboration in an agentic landscape. When an AI agent has pre-processed footage and flagged issues, the human-to-human review conversation changes. Design for that new interaction pattern, not the old one.

09File streaming vs. file downloading — why Frame.io Drive is a big deal technicallyFrame.io specific▼
The technical distinction
File downloading: The entire file is transferred to your local machine before you can use it. A 100GB 4K RAW file takes significant time to download on even a fast connection. You need local storage to hold it. And if the file changes on the server (new version uploaded), your local copy is now stale.

File streaming: The file stays on the server. Your local application reads it in real-time as it needs it — like streaming a Netflix video rather than downloading the whole movie first. Frame.io Drive uses Suite Studios' streaming technology to make local creative applications (Premiere, Photoshop, After Effects) believe they're reading local files when they're actually reading from Frame.io's cloud storage in real-time.

Local caching: The parts of the file you've already accessed get temporarily stored locally for performance — so the second time you scrub to a section of footage, it's already cached and plays back without latency.
Why this matters as a product concept
Frame.io Drive eliminates the "download and relink" problem that has plagued distributed film production for years. When a team of 5 editors in different cities are all working on the same production, traditionally each has to download copies of all the footage to their local machines — creating storage costs, version confusion, and file management overhead. With Frame.io Drive, they all work from the same cloud source in real-time. The collaboration happens at the storage layer, before they even open their NLE.

This is also why it's platform infrastructure: once an editor's NLE is configured to mount Frame.io as a local drive, every project they work on runs through Frame.io. The switching cost isn't "find a different review tool" — it's "restructure your entire editing workflow."

What a "pipeline" means in M&E
In film/TV/games production, a "pipeline" is the series of tools, processes, and data flows that move a production from raw materials to finished product. Unlike a simple linear workflow, a production pipeline is a network: VFX artists, compositors, colorists, and editors are all working in parallel on different shots, using different tools, and all feeding into the same final deliverable.

ShotGrid's role: ShotGrid (now Autodesk Flow Production Tracking) is the nervous system of that pipeline — it tracks every shot, every asset, every task, every version, every note, and every deadline across the entire network of parallel work. When a VFX shot is delivered for review in Frame.io, ShotGrid tracks that it's in review. When Frame.io marks it approved, ShotGrid updates the shot status. When the colorist picks it up next, they know it's cleared VFX review.

Why this matters for Frame.io: Victoria thinks about Frame.io as a node in a larger pipeline, not a standalone product. A PM who understands pipeline thinking will design integrations differently from one who only thinks about user experience in isolation.
The key pipeline concepts to know
Shot: A single continuous camera take. A feature film has thousands of shots. Each one is tracked through the pipeline from photography through VFX through color through delivery.

Asset: Any reusable element in a production — a 3D character model, a set piece, a sound effect. Assets feed into shots.

Version: Every iteration of a shot or asset delivered for review. VFX teams deliver V001, V002, V003 of a shot as they incorporate feedback. Frame.io's version stacking is designed for exactly this.

Deliverable vs. internal cut: An internal cut is a WIP review. A deliverable is the final, approved, client-facing or broadcast-ready asset. Frame.io serves both but the governance requirements are different.

CC / Creative Cloud — Adobe's subscription suite. Frame.io is included free with CC All Apps, Premiere Pro, and After Effects.
Firefly — Adobe's generative AI model. Text-to-image, generative fill, video generation. Trained on licensed content.
Firefly Services — API layer for bulk AI asset generation. Enables Reframe, Dub, Remove Background actions inside Frame.io.
Firefly AI Assistant — Launched 2026. Conversational agent orchestrating multi-step workflows across CC apps. Frame.io is the review/approval node.
GenStudio — Adobe's AI-first marketing content platform. Frame.io is embedded as the review/approval layer.
AEM — Adobe Experience Manager. Enterprise CMS and DAM. Where approved content goes after Frame.io review.
Workfront — Adobe's enterprise work management / PM platform. Separate org (Digital Experience) from Frame.io (Creative Cloud). Reports to different leadership.
Content Credentials — AI provenance standard. Attaches a verified record to each asset showing origin, edit history, AI usage. Persists when files are downloaded or shared.
CAI — Content Authenticity Initiative. Adobe-led industry coalition building the Content Credentials standard.
Admin Console — Adobe's enterprise account management portal. IT admins manage CC licenses, users, and Frame.io accounts here.
Workfront Fusion — Low-code/no-code automation platform within Workfront. Enables Frame.io ↔ Workfront integrations and 3,000+ other app connections.
Project Graph — In development. Node-based visual system for designing AI-powered workflows across CC. Frame.io is a node.
Platform + developer vocabulary — Victoria's domain
★API + developer platform termsHigh priority▼
Webhook — An event-triggered HTTP callback. When something happens in Frame.io (asset uploaded, approved, commented on), a webhook fires a payload to a URL you specify. Core building block of the Frame.io developer platform.
Custom Actions — Beta feature in Frame.io V4. Lets developers surface third-party logic directly inside Frame.io's UI. Right-click an asset → trigger an action that calls your external service.
REST API — Frame.io's V4 API uses REST (representational state transfer). Standard HTTP-based API pattern. V4 API managed through Adobe Developer Console, uses OAuth 2.0.
OAuth 2.0 — The authentication standard Frame.io V4 uses. Replaced the legacy developer tokens and JWT auth from V3.
SDK — Software Development Kit. Frame.io offers Python and TypeScript SDKs for the V4 API.
C2C Devices API — The specific API that enables hardware manufacturers to build Camera to Cloud into their devices. What Canon, Nikon, RED etc. integrate with.
MAM — Media Asset Management. Enterprise systems for storing, organizing, and retrieving large media files (different from DAM which is broader). Examples: Iconik, Evolphin, Dalet.
DAM — Digital Asset Management. Broader category including all digital assets. Examples: Bynder, Canto, Widen. AEM Assets is Adobe's DAM.
ShotGrid / Flow Production Tracking — Autodesk's production pipeline management tool. Victoria spent 6 years here. Tracks shots, assets, tasks for film/TV/games. Complementary to Frame.io, not competing.
Pipeline — In M&E context: the series of tools and workflows that move a production from pre-production through post to delivery. Victoria thinks about Frame.io as a pipeline tool.
Frame.io Drive — NAB 2026 announcement. Desktop app that mounts Frame.io cloud projects as a local drive in Finder/File Explorer. Built on Suite Studios file-streaming tech. Replaces Frame.io Transfer app.
Mounted storage — Making cloud storage appear as a local drive. What Frame.io Drive does. Key for editing workflows where NLEs need to read files as if they're local.
Business + product strategy terms
→PM + business vocabularyGood to know▼
PLG — Product-Led Growth. Growth driven by the product itself (free tiers, virality, self-serve). Frame.io's unlimited reviewers model and CC bundling are PLG mechanics.
ACV — Annual Contract Value. Revenue from a single customer per year. M&E enterprise = high ACV. SMB/freelancer = low ACV.
TAM — Total Addressable Market. Frame.io's narrow TAM (video review) ~$1.5B; broader creative ops TAM ~$8-12B+.
MAU / DAU — Monthly/Daily Active Users. Frame.io's MAU on CC-bundled base unknown publicly — itself a strategic issue.
ARR — Annual Recurring Revenue. Frame.io's standalone ARR estimated ~$15-25M at acquisition. Adobe doesn't break it out publicly.
P50 — Adobe's internal job leveling. Principal PM = P50. Senior PM = P40. The level you're interviewing for.
Content supply chain — Adobe's term for the end-to-end workflow from content brief → creation → review → delivery → activation → measurement. Frame.io sits in the review/approval layer.
NAB Show — National Association of Broadcasters. Major annual M&E industry conference in Las Vegas. Frame.io makes major announcements here (V4 beta NAB 2024, Drive NAB 2026).
Adobe MAX — Adobe's annual creativity conference. V4 GA launched October 2024. Major venue for CC product announcements.
Adobe Summit — Adobe's annual enterprise/marketing conference. GenStudio, Workfront, and Experience Cloud announcements. Where Adobe Brand Intelligence was announced (April 2026).
